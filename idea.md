# Poker Helper — 功能 Idea 记录

> 这个文件用来记录想法和设计决策，不是开发文档。代码实现以实际仓库为准。

## 1. Session 计时器 (优先级 P0)

### 核心需求
- 一个 section / session 默认 **3 小时**倒计时
- 打开页面看到一个 **Start** 按钮，点击 = 开始牌局，开始倒计时
- 倒计时归零时**响铃**提醒
- 提醒后可以：
  - **Stop**：停止响铃
  - **Restart**：重新计时（继续打）

### 中途操作
- **End** 按钮：随时结束当前 section（提前结束的场景）
  - 提前结束也要弹"记录"页面
- **延长**：到时间响铃后选 Restart，相当于又开始 3 小时；也可以考虑做"再延长 30 分钟"按钮
- 中途**离开页面 / 锁屏**：计时不能停。
  - 实现思路：存"开始时刻"的时间戳，每次进入页面用 `Date.now() - startTime` 算剩余，不用 setInterval 累加
  - 响铃如果在后台触发不了（iOS 限制），需要在打开页面那一刻立刻补上声音 + 红色横幅

### 结束后的录入页 (P0 配套)
弹出一个表单，填：
- **Buy-in 总额**（可能多次补码，先按总额填，后面再考虑分次记录）
- **Cash-out 金额**
- 自动带入：
  - 开始时刻
  - 结束时刻
  - 持续时长
- 提交后保存到云端，回到主页

### UX 草图
```

   ♠ Poker Helper
   ─────────────
   ▢ Start Session    (未在牌局时显示)

   [ End Session ]

   ─────────────
   最近 session:
     5/19  +$245  3h12m
     5/17  -$80   2h05m
     ...
```


## 2. 数据存储 (待定 → Supabase 倾向)

### 需求
- 跨设备同步（手机 / 电脑访问同一个 URL 都能看到自己的数据）
- 数据量小（每条 session 几十字节，一年也就几十 KB）
- 单用户为主（先做自己用）

### 候选
- **Supabase 免费档** ← 倾向这个
- Firebase Firestore（备选）
- localStorage（仅本机，不满足同步需求）
- GitHub API（写权限暴露问题，不可行）

### 数据模型 (草案)
```sql
-- sessions 表
id           uuid (主键)
user_id      uuid (Supabase 自带 auth.users 关联)
started_at   timestamptz
ended_at     timestamptz
buy_in       numeric        -- 美元 / 人民币 / 筹码？待定
cash_out     numeric
notes        text
created_at   timestamptz default now()
```

### 待定
- 货币单位（USD / RMB / chips？）
- 是否记录场地、牌局类型（NLH/PLO/...）、盲注大小
- 多次买入怎么处理（先合并成一个 buy-in 总额，够用就不细化）
